Beginner-friendly self-hosting

Two safe ways to deploy.

Choose guided help or run the commands yourself. Both paths use the same reviewed scripts, dry runs and confirmation gates.

  • No VPS or DockerRuns on Cloudflare
  • Free-tier friendlyPay only if usage grows

Before you start

You need a GitHub account, a Cloudflare account and one hostname such as go.example.com. Cloudflare Pages supplies the Admin URL automatically.

A · Recommended for beginners

Deploy with an AI assistant

Open your fork in Codex, Claude Code or Cursor, then give it the guarded prompt below. It will explain each choice and pause before any Cloudflare or GitHub write.

  1. 1

    Fork Linketry into your own GitHub account.

  2. 2

    Clone your fork and open the project in your AI assistant.

  3. 3

    Copy this prompt and follow the assistant's checkpoints.

Safe deployment prompt 
Help me deploy Linketry from my fork to my own Cloudflare account.

1. Read AGENTS.md and docs/SELF_HOSTING.md before acting.
2. Use the basic fresh-install profile: one Worker hostname, D1, KV and the automatic Pages Admin URL. Do not enable optional R2, Queue or branded Admin DNS.
3. Run deploy:bootstrap and deploy:configure in dry-run mode first. Show me the resource plan and exact confirmation phrase before every apply or deployment.
4. Keep Cloudflare and GitHub tokens out of source files, logs and command arguments. Ask me to enter them only through supported hidden prompts or secret stores.
5. Stop on any safety-gate failure. Do not bypass migration, account or domain checks.
6. After deployment, verify /health, the Admin login page and one test short link.

The prompt never asks you to paste a token into chat or commit it to the repository.

B · Guided command line

Run the maintained workflow yourself

Best when you already use GitHub CLI and Wrangler. Every resource is previewed before it is created or connected.

  1. 1

    Install dependencies and sign in to Cloudflare.

  2. 2

    Preview, review and create only the required D1 and KV.

  3. 3

    Preview repository settings, then start the protected workflow.

npm ci
npx wrangler login
npm run deploy:bootstrap -- --prefix linketry-you --domain go.example.com --account-id <account-id>

# Review the dry-run, then repeat it with the printed
# --apply --confirm phrase.

$repo = 'OWNER/REPOSITORY'
gh secret set CLOUDFLARE_API_TOKEN --repo $repo
gh api --method PUT "repos/$repo/environments/production"
npm run deploy:configure -- --repo $repo --prefix linketry-you --domain go.example.com --account-id <account-id>

# Review this dry-run, then repeat it with the printed
# --apply --confirm phrase before starting deployment.
gh workflow run deploy.yml --repo $repo --ref main --field confirm_release=true
